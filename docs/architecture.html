Flow

Data Flow — Analysis Request

Complete request lifecycle from user input to displayed results
1
Dashboard — User enters URL, selects modules/provider, clicks Analyze
2
Frontend POST — Next.js POSTs to /analyses on FastAPI (port 8000) with target_url, modules[], provider, fmt
3
API validates — Checks API key, module names against VALID_MODULES, creates job in SQLite (status=pending)
4
Background thread — worker.start_analysis_job() launches daemon thread, sets status=running
5
Core services — run_analysis() resolves LLM provider via llm.router, calls coordinator.run_all()
6
Coordinator — Executes modules in dependency order with progress callbacks (pct 5→100)
7
Report written — report.writer generates md/html file on disk
8
DB updated — Worker sets status=completed, results=..., report_path=...
9
Frontend polls — GET /analyses/{id}/results every 3s, displays results when completed
10
User views — Score badge on card, full breakdown on detail page, download report
